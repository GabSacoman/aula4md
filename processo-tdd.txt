O processo de Desenvolvimento Orientado a Testes (TDD) é um ciclo constante, no qual você começa escrevendo um teste funcional para a funcionalidade desejada, em seguida, escreve testes unitários que cobrem as funcionalidades exigidas pelo teste funcional. Você sempre deve escrever a menor quantidade de código possível para garantir que os testes unitários funcionem corretamente antes de se concentrar em fazer o teste funcional funcionar. Em seguida, você verifica se o código precisa ser refatorado.
